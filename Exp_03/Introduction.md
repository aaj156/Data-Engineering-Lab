# Relational Schema Design & SQL Normalization using PostgreSQL

> **Experiment No 03**

Design a normalized relational database from a real-world problem statement, implement it in PostgreSQL, perform analytical SQL queries, and optimize query performance using modern database techniques.

---

## 🎯 Experiment Objective

In this experiment, you will learn how to transform a real-world business problem into a well-designed relational database.

The experiment covers the complete database design lifecycle, including:

- Requirement Analysis
- Entity Relationship (ER) Modeling
- Database Normalization (1NF → 3NF)
- Relational Schema Design
- PostgreSQL Database Implementation
- Data Insertion using SQL
- Analytical SQL Queries
- SQL Window Functions
- Query Optimization
- OLAP-style SQL Analytics

---

## 🎓 Learning Outcomes

After completing this experiment, you will be able to:

- Design an Entity Relationship (ER) model from a problem statement.
- Apply database normalization up to Third Normal Form (3NF).
- Implement a normalized relational schema in PostgreSQL.
- Populate database tables using SQL DML commands.
- Write SQL queries using joins, grouping, aggregation, and subqueries.
- Perform analytical queries using SQL window functions.
- Optimize SQL queries using indexes, partitions, and execution plans.
- Develop OLAP-style analytical reports using SQL.
